# 📖 دليل البروتوكول التفصيلي

## هيكل الرسالة الأساسي

كل رسالة تبدأ بـ `IW` وتنتهي بـ `#`:
```
IW[COMMAND][DATA]#
```

## 📥 رسائل من الساعة للسيرفر (AP)

### 1. تسجيل الدخول (AP00)

**الرسالة:**
```
IWAP00353456789012345#
```

**المكونات:**
- `IW` - معرّف ثابت
- `AP00` - أمر تسجيل الدخول
- `353456789012345` - IMEI (15 رقم)
- `#` - نهاية الرسالة

**الرد المتوقع:**
```
IWBP00,20250330123456,3#
```
- الوقت الحالي بصيغة UTC
- المنطقة الزمنية (+3 للأردن)

---

### 2. موقع GPS (AP01)

**الرسالة:**
```
IWAP01080524A2232.9806N11404.9355E000.1061830323.8706000908000102,460,0,9520,3671,Home|74-DE-2B-44-88-8C|97#
```

**تحليل المكونات:**

**التاريخ والوقت:**
- `080524` - 24 مايو 2008
- `061830` - الساعة 06:18:30 GMT

**GPS:**
- `A` - بيانات صحيحة (V = غير صحيحة)
- `2232.9806N` - خط العرض: 22° 32.9806' شمالاً
- `11404.9355E` - خط الطول: 114° 04.9355' شرقاً
- `000.1` - السرعة (كم/ساعة)
- `323.87` - الاتجاه (درجات)

**الحالة:**
- `060` - إشارة GSM (0-100)
- `009` - عدد الأقمار الصناعية
- `080` - مستوى البطارية (%)
- `0` - المساحة المتبقية
- `01` - حالة التحصين
- `02` - وضع العمل

**LBS (أبراج الشبكة):**
- `460` - MCC (رمز الدولة - الصين)
- `0` - MNC (رمز المشغل)
- `9520` - LAC (Location Area Code)
- `3671` - Cell ID

**WIFI:**
- `Home|74-DE-2B-44-88-8C|97` - SSID|MAC|Signal
- يمكن أن يحتوي على عدة شبكات مفصولة بـ `&`

**الرد:**
```
IWBP01#
```

---

### 3. Heartbeat (AP03)

**الرسالة:**
```
IWAP03,06000908000102,5555,30#
```

**المكونات:**
- نفس بيانات الحالة من AP01
- `5555` - عدد الخطوات
- `30` - تكرار الحركة

**الرد:**
```
IWBP03#
```

---

### 4. إنذار (AP10)

**الرسالة:**
```
IWAP10080524A2232.9806N11404.9355E...00,zh-cn,00,WIFI#
```

**مشابه لـ AP01 مع إضافة:**
- `00` - نوع الإنذار:
  - `01` - SOS
  - `03` - خلع الساعة
  - `05`/`06` - وقوع
- `zh-cn` - لغة الجهاز

**الرد:**
```
IWBP10#
```

أو مع عنوان:
```
IWBP106df157335e02...#
```

---

### 5. قياسات صحية

#### نبض القلب (AP49)
```
IWAP49,68#
```
- `68` - نبض القلب (bpm)

#### نبض + ضغط (APHT)
```
IWAPHT,60,130,85#
```
- `60` - نبض
- `130` - ضغط انقباضي
- `85` - ضغط انبساطي

#### قياسات كاملة (APHP)
```
IWAPHP,60,130,85,95,90,,,,,,,,#
```
- `60` - نبض
- `130` - ضغط انقباضي
- `85` - ضغط انبساطي
- `95` - أكسجين (SPO2)
- `90` - سكر الدم

#### حرارة (AP50)
```
IWAP50,36.7,90#
```
- `36.7` - حرارة الجسم (°C)
- `90` - البطارية (%)

## 📤 أوامر من السيرفر للساعة (BP)

### 1. طلب موقع فوري (BP16)

**الأمر:**
```
IWBP16,353456789012345,080835#
```
- IMEI
- Journal Number (رقم السجل)

**الرد المتوقع:**
```
IWAP16,080835#
```
ثم ترسل الساعة AP01 بالموقع

---

### 2. تفعيل القياسات

**نبض القلب (BPXL):**
```
IWBPXL,353456789012345,080835#
```

**ضغط الدم (BPXY):**
```
IWBPXY,353456789012345,080835#
```

**الحرارة (BPXT):**
```
IWBPXT,353456789012345,080835#
```

**الأكسجين (BPXZ):**
```
IWBPXZ,353456789012345,080835#
```

---

### 3. ضبط أرقام SOS (BP12)

**الأمر:**
```
IWBP12,353456789012345,080835,135XXXXXXXX,135XXXXXXXX,135XXXXXXXX#
```
- 3 أرقام كحد أقصى
- إذا كان رقم فارغ، اترك فراغ

**الرد:**
```
IWAP12,080835,135XXXXXXXX,135XXXXXXXX,135XXXXXXXX#
```

---

### 4. ضبط وضع العمل (BP33)

**الأمر:**
```
IWBP33,353456789012345,080835,1#
```

**الأوضاع:**
- `1` - عادي: موقع كل 15 دقيقة
- `2` - توفير طاقة: موقع كل 60 دقيقة
- `3` - طوارئ: موقع كل 1 دقيقة

**الرد:**
```
IWAP33,080835,1#
```

---

### 5. كشف الوقوع (BP76)

**تفعيل/إيقاف:**
```
IWBP76,353456789012345,080835,1#
```
- `1` - تفعيل
- `0` - إيقاف

**ضبط الحساسية (BP77):**
```
IWBP77,353456789012345,080835,3#
```
- `1` - منخفضة
- `2` - متوسطة
- `3` - عالية (الأكثر حساسية)

---

### 6. ضبط المنطقة الزمنية (BP20)

```
IWBP20,353456789012345,080835,0,3#
```
- `0` - اللغة (غير مستخدم)
- `3` - المنطقة الزمنية (+3 للأردن)

---

### 7. ضبط فترة قياس النبض التلقائي (BP86)

```
IWBP86,353456789012345,080835,1,720#
```
- `1` - تفعيل (0 للإيقاف)
- `720` - كل 720 دقيقة (12 ساعة)

---

### 8. إعادة ضبط المصنع (BP17)

```
IWBP17,353456789012345,080835#
```

⚠️ **تحذير:** هذا سيمسح جميع الإعدادات!

---

### 9. إيقاف التشغيل (BP31)

```
IWBP31,353456789012345,080835#
```

## 🔢 رقم السجل (Journal Number)

رقم السجل هو معرّف للأمر، صيغته:
```
MMDDHHMMSS
```
- `MM` - الشهر
- `DD` - اليوم
- `HH` - الساعة
- `MM` - الدقيقة
- `SS` - الثانية

مثال: `0330123045` = 30 مارس، الساعة 12:30:45

---

## 📊 تحويل الإحداثيات

الساعة ترسل الإحداثيات بصيغة **NMEA**:
```
2232.9806N = 22° 32.9806'
```

**التحويل إلى Decimal:**
```
Degrees = 22
Minutes = 32.9806
Decimal = 22 + (32.9806 / 60) = 22.549677°
```

**إذا كان الاتجاه S أو W، اجعل القيمة سالبة.**

## 🎨 مثال كامل: تدفق الاتصال

```
1. الساعة تتصل:
   → IWAP00353456789012345#

2. السيرفر يرد:
   ← IWBP00,20250330123456,3#

3. الساعة ترسل موقع:
   → IWAP01080524A2232.9806N11404.9355E...#

4. السيرفر يرد:
   ← IWBP01#

5. السيرفر يطلب نبض:
   ← IWBPXL,353456789012345,123456#

6. الساعة ترد:
   → IWAPXL,123456#

7. الساعة ترسل النبض:
   → IWAP49,72#

8. السيرفر يرد:
   ← IWBP49#
```

## 🔐 ملاحظات أمنية

1. **لا يوجد تشفير في البروتوكول الأساسي**
   - استخدم VPN أو SSL/TLS Tunnel للإنتاج

2. **لا يوجد مصادقة قوية**
   - IMEI يمكن تزويره نظرياً
   - ضع Firewall Rules

3. **البيانات الحساسة**
   - القياسات الصحية حساسة
   - تأكد من تشفير قاعدة البيانات

## 🧪 اختبار البروتوكول

يمكنك اختبار السيرفر باستخدام `netcat`:

```bash
# الاتصال بالسيرفر
nc localhost 5088

# إرسال رسالة تسجيل دخول
IWAP00353456789012345#

# انتظر الرد من السيرفر
```

أو استخدام Python:
```python
import socket

sock = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
sock.connect(('localhost', 5088))

# إرسال login
sock.send(b'IWAP00353456789012345#')

# استقبال الرد
response = sock.recv(1024)
print(response.decode())

sock.close()
```
